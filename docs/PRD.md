# Ember — 双人书信日记 产品需求文档

## 1. 产品概述

Ember 是一款面向情侣/密友的双人书信式日记 Web 应用。两位用户配对后，可以像写信一样给对方写日记，对方收到后可以阅读和回信。支持移动端优先的响应式设计。

## 2. 目标用户

- 情侣、密友等两人关系
- 希望用文字记录和分享日常感受

## 3. 核心功能

### 3.1 账号系统

| 功能 | 说明 |
|------|------|
| 邮箱注册 | 邮箱 + 密码注册，发送验证邮件激活 |
| 登录 | 邮箱 + 密码登录 |
| 忘记密码 | 通过邮箱发送重置链接 |
| 个人资料 | 昵称、头像 |

### 3.2 配对机制

- 注册后生成唯一邀请码
- 将邀请码分享给对方，对方输入后完成配对
- 一个账号同一时间只能与一人配对
- 支持解除配对（需双方确认或单方解除）

### 3.3 书信日记

| 功能 | 说明 |
|------|------|
| 写信 | 富文本编辑器，支持文字、图片、emoji |
| 发送 | 写完后发送给对方 |
| 收信 | 收到对方的信，按时间线展示 |
| 草稿箱 | 未发送的信自动保存为草稿 |
| 时间线 | 双方的信按时间交替排列，类似聊天记录但以信件形式呈现 |
| 信件详情 | 展开阅读完整内容，显示写信时间 |

### 3.4 通知

- 收到新信时通过邮件通知
- 应用内未读标记

### 3.5 其他

- 日记支持设置"心情标签"（开心、难过、思念等）
- 纪念日统计：配对天数、总信件数

## 4. 非功能需求

| 项目 | 要求 |
|------|------|
| 响应式 | 移动端优先，适配 375px~1440px |
| 性能 | 首屏加载 < 2s |
| 安全 | HTTPS、密码加密存储、邮件验证 |
| 隐私 | 日记内容端对端仅双方可见，管理员不可读 |

## 5. 技术方案（建议）

| 层 | 技术 |
|----|------|
| 前端 | Next.js (App Router) + Tailwind CSS |
| 后端 | Next.js API Routes / Route Handlers |
| 数据库 | PostgreSQL (via Supabase 或 Prisma + 自建) |
| 认证 | NextAuth.js（Credentials Provider + 邮箱验证） |
| 文件存储 | Cloudflare R2 / AWS S3（图片上传） |
| 邮件 | Resend / SendGrid |
| 部署 | Vercel |

## 6. 页面清单

| 页面 | 路由 | 说明 |
|------|------|------|
| 首页/落地页 | `/` | 产品介绍、登录/注册入口 |
| 注册 | `/signup` | 邮箱注册 |
| 登录 | `/login` | 邮箱登录 |
| 忘记密码 | `/forgot-password` | 发送重置邮件 |
| 重置密码 | `/reset-password` | 设置新密码 |
| 配对 | `/pair` | 输入/分享邀请码 |
| 时间线 | `/timeline` | 双方信件时间线（主页面） |
| 写信 | `/compose` | 编写新信件 |
| 信件详情 | `/letter/[id]` | 查看单封信件 |
| 个人设置 | `/settings` | 昵称、头像、密码修改、解除配对 |

## 7. 数据模型（概要）

```
User: id, email, password_hash, nickname, avatar_url, invite_code, paired_with, created_at
Letter: id, author_id, recipient_id, title, content, mood, is_draft, sent_at, read_at, created_at
```

## 8. MVP 范围

第一期只做：

1. 邮箱注册/登录/重置密码
2. 邀请码配对
3. 写信、发送、收信、时间线浏览
4. 移动端适配
5. 邮件通知（收到新信）

后续迭代：
- 图片上传
- 心情标签
- 纪念日统计
- PWA 离线支持
